THE 201

You've used AI to write copy.
Now use it to build tools.

Claude Code is a CLI-based AI developer that lives in your terminal. It doesn't just generate text — it writes code, connects to your systems, builds dashboards, and deploys them. In plain English.

See the difference
claude — ~/marketing-ops

CHAT AI (what you know)

Generates text in a browser

  • You paste data in, get text back
  • Can't connect to your actual systems
  • Outputs live in a chat window
  • Each conversation starts from scratch
  • You copy-paste results into other tools
→

CLI AI (the paradigm shift)

Builds software on your machine

  • Reads and writes files directly on your computer
  • Connects to Salesforce, databases, APIs live
  • Outputs are real apps, scripts, and dashboards
  • CLAUDE.md gives it persistent project memory
  • Deploys working tools your team can use

Subagent Architecture — How It Works
Parallel Subagent Workflow
MAIN AGENT Orchestrator

Receives your request, breaks it into independent research tasks, and dispatches subagents in parallel. Synthesizes results when all return.

AGENT 1

Salesforce MCP → pipeline data, win rates, deal velocity

AGENT 2

GA4 + Ads API → traffic, conversions, ROAS by channel

AGENT 3

Sprout API → engagement, follower growth, top content

MAIN AGENT Synthesis

Combines all data into a cohesive deck with cross-channel insights. Each agent ran independently — total time ~12 seconds, not 3 minutes sequential.

This is the real power. Complex multi-source tasks that would take hours manually run in seconds with parallel subagents.
How to Think About It

The mental models that matter

Using Claude Code well isn't about prompt tricks. It's about understanding how it works under the hood — especially context management.

🧠

Your context window is finite. Treat it like RAM.

THE SINGLE MOST IMPORTANT CONCEPT

Claude Code has a context window — roughly 200K tokens. That sounds like a lot, but it fills up fast when you're reading files, querying APIs, and iterating on code. When it fills up, Claude compresses older messages and loses nuance. Your job is to keep the context clean so the AI stays sharp.

Context Window Usage ~40% — Fresh & focused
0% — Start 50% — Peak focus 75% — Delegate or /clear 100% — Compacting
The trap: Don't dump your entire Salesforce schema, all your campaign data, AND your brand guidelines into one session. Load only what the current task needs. Use subagents for independent research so the main window stays clean.
🔌

MCP: Your data layer

MODEL CONTEXT PROTOCOL

MCP servers are how Claude Code talks to external systems — Salesforce, databases, Google APIs, Slack. Think of them as typed adapters: Claude knows what tools are available and what data it can pull. Set up once per project.

// .mcp.json — project config
{
  "salesforce": {
    "command": "npx",
    "args": ["sf-mcp-server"],
    "env": { "SF_TOKEN": "${SF_TOKEN}" }
  }
}
Context cost: Each MCP server's tool definitions eat context tokens just by existing. Only connect what you actually need per project. 3 focused servers > 10 "just in case" ones.
🔀

Subagents: Parallel workers

DELEGATION, NOT SEQUENCE

When a task involves gathering data from multiple sources or doing independent work, Claude Code can spawn subagents — lightweight workers that run in parallel and return results to the main agent.

This is powerful for two reasons: speed (parallel > sequential) and context hygiene (subagent work doesn't bloat your main window).

When to delegate: If you need info from 2+ independent sources, or if a subtask doesn't need your conversation history to make sense, it should be a subagent.
📋

CLAUDE.md: Project memory

PERSISTENT INSTRUCTIONS

Every session starts fresh — Claude Code doesn't remember yesterday's session. But it always reads CLAUDE.md from your project directory. This is where you encode your brand voice, data conventions, preferred tools, and team processes.

# CLAUDE.md
## Brand
Colors: Navy #1a1a2e, Teal #00d4aa
Voice: Professional, warm, concise

## Salesforce
Org: production (not sandbox)
Key objects: Lead, Opportunity, Campaign
Custom fields: Lead_Score__c, ICP_Fit__c
🧹

Know when to /clear

FRESH CONTEXT = BETTER OUTPUT

After finishing a task, start a new context for the next one. Leftover context from building a dashboard will confuse the AI when you switch to writing email templates. Think of each /clear as closing tabs in your browser.

Rule of thumb: One task per session. Dashboard? /clear. Email templates? /clear. Report? /clear. Your CLAUDE.md carries the important context forward automatically.
🔁

Iterate, don't over-specify upfront

BUILD → SEE → REFINE

Don't try to write the perfect prompt. Give Claude Code the core requirements, let it build a first pass, then refine. "Make the chart bigger." "Switch to a bar chart." "Add last quarter for comparison." This is faster and produces better results than trying to specify everything in one shot.

Why this works: Each iteration adds ~200 tokens of context. A massive upfront prompt with every detail adds thousands — and you'll still need to iterate anyway.
🎯

Specificity beats length

SIGNAL OVER NOISE

A 3-line prompt with the right details outperforms a 30-line prompt that's vague. Name your data sources. Name your output format. Name your audience. Claude Code works best when it knows exactly what "done" looks like.

# Vague — Claude has to guess
> Build me a marketing report

# Specific — Claude nails it
> Build an HTML report showing Q1
  Google Ads spend vs Salesforce
  closed-won revenue by campaign.
  Include a ROAS column. Export to PDF.
