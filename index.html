Getting the Most Out of It

Best practices

Tips from power users to help you get better results faster.

💬

Be specific in your prompts

"Build a dashboard" is okay. "Build a dashboard showing Q1 pipeline by rep with a filter for deal stage and a CSV export button" is much better. Context is everything.

🔌

Connect your tools via MCP

Model Context Protocol lets Claude Code talk directly to Salesforce, databases, APIs, and more. Set it up once, use it forever. This is where the real power is.

🔁

Iterate, don't restart

Claude Code remembers context within a session. Instead of starting over, refine: "Make it wider." "Add last month's data." "Change the colors to match our brand." Build on what's there.

📋

Use CLAUDE.md for project memory

Drop a CLAUDE.md file in your project with brand guidelines, data conventions, and tool preferences. Claude Code reads it automatically and follows your rules every session.

📐

Start small, expand

Get a working prototype first, then add complexity. "Build the basic chart" → "Add filters" → "Add drill-down" → "Schedule a daily email." Incremental wins compound.

🧪

Let it run tests

Claude Code can write and run its own tests. When building automations that touch live data (especially Salesforce), ask it to test against sandbox data first.
